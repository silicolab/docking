# docking

A pure-Rust reimplementation of the [AutoDock Vina](https://vina.scripps.edu/)
molecular docking engine.

`docking` is an **engine-only** crate: given an already-prepared receptor and
ligand (PDBQT) plus a search box, it produces ranked docked poses with Vina
scores. It performs no ligand/receptor *preparation* (protonation, charge
assignment, rotatable-bond detection) — that is expected to be done upstream by
tools such as Meeko / AutoDockTools.

## Usage

The high-level entry point is the `api` module:

```rust
use docking::api::{dock, DockConfig};

let receptor = std::fs::read_to_string("receptor.pdbqt")?;
let ligand = std::fs::read_to_string("ligand.pdbqt")?;

let poses = dock(
    &receptor,
    &ligand,
    [15.190, 53.903, 16.917], // box center (Å)
    [20.0, 20.0, 20.0],       // box size (Å)
    &DockConfig { seed: 42, ..Default::default() },
)?;

for (rank, pose) in poses.iter().enumerate() {
    println!("mode {}: {:.3} kcal/mol", rank + 1, pose.affinity);
    // pose.pdbqt is the docked conformation, ready to write out.
}
# Ok::<(), Box<dyn std::error::Error>>(())
```

`api::score_only` evaluates a single input pose (Vina's `--score_only`).

## Fidelity

The top priority of this port is to reproduce official Vina's results. The
fidelity strategy is:

- **Deterministic math** (scoring terms, conformation, analytic gradients,
  grid interpolation) is validated to be numerically equivalent to official
  Vina, term by term, within tight tolerances.
- **The Monte Carlo global search trajectory** is validated by *statistical
  equivalence and convergence*: with a fixed seed the search must converge to
  the same energy minima / pose clusters as official Vina within a documented
  tolerance, rather than reproducing Boost's exact RNG byte stream.

Ground-truth poses and scores are generated by the official Vina **v1.2.7**
binary and archived under `tests/golden/`. A diff harness compares Rust output
against these golden values with explicit tolerances.
